Administration¶

Ajouté dans la version 0.3.0.

Application¶

Seuls les utilisateurs disposant de droits d'administration peuvent accéder à l'administration de l'application.

Configuration¶

Modifié dans la version 0.6.0: ajout de l'adresse électronique pour contacter l'administrateur

Modifié dans la version 0.7.4: ajout de la taille maximale du fichier et du nombre maximale de fichiers dans une archive

Modifié dans la version 0.8.5: ajout du nombre maximum de séances pour les statistiques d'un sport

Modifié dans la version 0.10.0: ajout du nombre maximal de fichier pour l'ajout synchrone de fichiers et du nombre maximal des séances affichées sur la carte globale

Les paramètres suivants peuvent être définis :

  • limite d'utilisateurs actifs (valeur par défaut : 0). Si la valeur est égale à 0, l'inscription est activée (aucune limite n'est définie).

  • taille maximale du fichier (ajouté unitairement ou dans une archive zip, valeur par défaut : 1 Mo)

  • taille maximale de l'archive zip (valeur par défaut : 10 Mo)

  • nombre maximum de fichiers dans l'archive zip (valeur par défaut : 10)

  • nombre maximum de fichiers pour le traitement synchrone (valeur par défaut : 10). Si le nombre maximum de fichiers dans l'archive zip est égal à celui des fichiers traités en synchrone, le chargement asynchrone est désactivé.

Note

Lors de la mise à jour vers la version 0.10.0, le téléchargement asynchrone est désactivé, puisque les deux valeurs sont égales.

  • nombre maximum de séances pour les statistiques d'un sport (par défaut : 10000). S'il est égal à 0, toutes les séances sont récupérées pour calculer les statistiques

  • nombre maximum de séances affichées sur la carte globale (par défaut : 10000), cette valeur ne peut pas dépasser 50000

Note

Le nombre maximal de séances pour les statistiques ou la carte globale doit être défini en fonction des capacités du serveur.
Dans le cas de la carte globale, cette valeur a également un impact sur les performances de rendu côté client (en fonction du navigateur et des capacités de l'appareil affichant la carte).

  • adresse électronique pour contacter l'administrateur

Avertissement

Si plusieurs workers pour l'application sont lancés (voir la variables d'environnement), il peut être nécessaire de tous les redémarrer pour prendre en compte les changements.
La mise à jour du timeout (voir la variable d'environnement) ou de la configuration du serveur peut être nécessaire pour gérer des fichiers volumineux (comme nginx par exemple).
Des valeurs trop élevées pour la taille ou le nombre de fichiers peuvent être entraîner des erreurs.
Si un fournisseur de données météo est configuré, des erreurs liées à la limitation de l'API peuvent être rencontrées.

Note

Si l'envoi de courriels est désactivé, un avertissement s'affiche.

A propos¶

Ajouté dans la version 0.7.13.

Il est possible d'afficher des informations supplémentaires qui peuvent être utiles aux utilisateurs dans la page A propos (comme les règles de l'instance).
La syntaxe Markdown peut être utilisée.

Politique de confidentialité¶

Ajouté dans la version 0.7.13.

Une politique de confidentialité par défaut est disponible (adaptée de la Politique de confidentialité de Discourse).
Une politique de confidentialité personnalisée peut être définie si nécessaire (la syntaxe Markdown peut être utilisée). Une mise à jour de la politique affichera un message sur le tableau de bord des utilisateurs pour qu'ils puissent en prendre connaissance.

Note

Seule la politique de confidentialité par défaut est traduite (si la traduction est disponible).

Modération¶

Ajouté dans la version 0.9.0.

Seul l'utilisateur ayant des droits d'administration peut modifier un autre utilisateur.

This page displays the list of reports, which can be filtered.

Administrators and moderators can manage a report:

  • ajouter un commentaire

  • envoyer un avertissement

  • suspendre ou réactiver une séance ou un commentaire

  • suspendre ou réactiver le compte d'un utilisateur

  • marquer un signalement comme résolu ou non résolu

Note

Le contenu signalé est visible quelque soit le niveau de visibilité.

Les utilisateurs peuvent faire appel de la suspension ou de l'avertissement.
L'utilisateur suspendu peut uniquement accéder à son compte, faire appel de la suspension du compte, demander l'export de données ou supprimer son compte. Ses séances et commentaires ne sont plus visibles.

Types d'équipement¶

Ajouté dans la version 0.8.0.

Seuls les utilisateurs disposant de droits d'administration peuvent accéder l'administration des types d'équipement.

Il est possible d'activer ou désactiver un type d'équipement afin de correspondre aux sports désactivés (un type d'équipement peut être désactivé même si un équipement pour ce type existe).

Tâches en attente¶

Ajouté dans la version 0.10.0.

Seuls les utilisateurs ayant des droits d'administration peuvent accéder aux tâches en attente pour les exports des données utilisateurs ou le chargement des archives de séances.

Note

Si aucun worker n'est lancé, une interface de ligne de commandes permet de traiter les tâches en attente.

Sports¶

Seuls les utilisateurs ayant des droits d'administration peuvent accéder à l'administration des sports.
Il est possible d'activer ou désactiver un sport (un sport peut être désactivé même si une séance avec ce sport existe).

Utilisateurs¶

Modifié dans la version 0.6.0: ajout de l'activation d'un compte utilisateur et de la mise à jour de l'adresse électronique et du mot de passe

Modifié dans la version 0.9.0: ajout des rôles de modérateur et propriétaire, mise à jour du rôle d'administrateur

Seuls les utilisateurs ayant des droits d'administration peuvent accéder à l'administration des utilisateurs.

Les rôles suivants sont disponibles :

  • utilisateur

    • pas de droits de modération ou d'administration

  • modérateur

    • peut seulement accéder à l'entrée modération dans l'administration

    • peut voir les signalements

    • effectuer des actions liées aux signalements

  • administrateur

    • a les droits de modération

    • peut accéder à toutes les entrées de l'administration :

      • application

      • modération

      • types d'équipement

      • sports

      • utilisateurs

  • propriétaire

    • a les droits d'administration

    • ce rôle ne peut pas être modifié par un autre administrateur ou propriétaire dans l'application

Note

Les rôles définis avant la version 0.9.0 restent inchangés.

L'administration des utilisateurs affiche la liste des utilisateurs, qui peut être filtrée.

Il est possible de modifier un utilisateur afin de :

  • mettre à jour le rôle. Un utilisateur avec un rôle de propriétaire ne peut pas être modifié par un autre utilisateur. Le rôle de propriétaire ne peut être assigné ou retiré qu'avec la CLI de FitTrackee.

  • activer son compte

  • mettre à jour son adresse électronique (au cas où son compte serait bloqué)

  • réinitialiser son mot de passe (dans le cas où son compte est verrouillé).Si l'envoi des courriels est désactivé, cela n'est possible que via l'interface de ligne de commande (CLI).

Il est également possible le compte d'un utilisateur.
